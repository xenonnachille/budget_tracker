# Budget Tracker (GitHub Pages + Offline + Google Sheets Sync)

Готовое PWA-приложение для учёта **доходов и расходов**:

- работает оффлайн (Service Worker + localStorage),
- автоматически синхронизируется при появлении интернета,
- поддерживает свои категории,
- предлагает частые категории автоматически,
- адаптировано под Android и Windows (responsive + installable PWA).

## 1) Публикация на GitHub Pages

1. Создайте репозиторий на GitHub.
2. Загрузите файлы этого проекта.
3. В GitHub: **Settings → Pages**.
4. Включите публикацию из ветки (`main`/`master`) и корня (`/root`).
5. Получите URL вида `https://<username>.github.io/<repo>/`.

> Если деплой в подпапку репозитория, оставляйте `start_url: "./"` в `manifest.json` как сейчас.

## 2) Подключение Google Sheets для синхронизации

### Шаг A. Подготовка таблицы

1. Создайте Google Spreadsheet.
2. Откройте **Extensions → Apps Script**.
3. Вставьте код из файла `apps-script.gs`.
4. (Рекомендуется) замените `TOKEN` на свой секрет.

### Шаг B. Публикация Apps Script как Web App

1. **Deploy → New deployment**.
2. Тип: **Web app**.
3. Execute as: **Me**.
4. Who has access: **Anyone** (или по вашей политике доступа).
5. Скопируйте `Web app URL`.

### Шаг C. Подключение в приложении

1. Откройте ваш сайт на GitHub Pages.
2. В блоке **Настройка Google Sheets** вставьте `Web app URL`.
3. Введите тот же токен, если включили его.
4. Нажмите **Сохранить настройки**.

## Как работает синхронизация

- При добавлении операции запись сохраняется локально сразу.
- Если сеть есть — запускается push/pull с Google Sheets.
- Если сети нет — данные остаются локально и отправятся автоматически после события `online`.
- Дополнительно приложение делает периодическую попытку синхронизации раз в минуту.

## Структура

- `index.html` — интерфейс приложения
- `styles.css` — адаптивные стили
- `app.js` — логика бюджета, категорий, PWA, sync
- `sw.js` — оффлайн-кэш
- `manifest.json` — PWA-манифест
- `apps-script.gs` — серверная часть синхронизации для Google Apps Script

## Локальный запуск

```bash
python3 -m http.server 8000
```

Откройте: `http://localhost:8000`

## Если видите "Синхронизация: Ошибка синхронизации"

Проверьте:

1. URL вставлен полностью и заканчивается на `/exec` (а не `/dev`).
2. Web App задеплоен с доступом `Anyone` (или совместимым с вашим сценарием).
3. Если задан `TOKEN`, он совпадает в приложении и в `apps-script.gs`.

В клиенте используется CORS-совместимый формат POST (`text/plain`), чтобы избежать preflight-проблем в браузерах.
